Qué, porqué y para qué

El desarrollo de este proyecto se basa en el diseño e implementación del proceso de desarrollo Iterativo e Incremental acorde con las metodologías ágiles a través de la forja SidelabCode Stack.

Se busca unificar el comportamiento de las herramientas a través de las distintas APIs, facilitar la instalación, la replicación y la recuperación de los datos mediante el uso de Software Libre para construir Software de calidad.

Para comenzar formularemos tres preguntas: que, por que,y para que

Procesos de Desarrollo

Actúan de Catalizadores

Los procesos de desarrollo son el siguiente requerimiento. Actúan de catalizadores del mismo desarrollo de software. Vamos a adaptar los siguientes...

TDD

Proceso de desarrollo guiado por pruebas.

El TDD se basa en la repetición de un ciclo de desarrollo corto en el que:

Punto Rojo - El desarrollador escribe un caso de prueba que define el comportamiento que se busca en la nueva funcionalidad y ha de fallar.

Punto Verde - A partir de este instante ya están definidos los requerimientos de la funcionalidad en caso de prueba y se ha escribir el código necesario para que a través del test se obtenga el resultado esperado.

Punto Morado - Acto seguido se ha de refactorizar el código y se da por terminada la tarea.

Feature branch

Desarrollo dirigido por ramas (M. Fowler)

El Desarrollo por ramas ayuda a la creación de código de una forma ordenada y coordinada dentro de un grupo de trabajo. Se basa en la creación y la integración continua de nuevas funcionalidades o solventar problemas derivados de una manera eficiente, ágil y distribuida.

El desarrollo basado en ramas dentro del proceso de desarrollo Iterativo e Incremental ayuda a la integración del código en la rama principal de desarrollo. Cada iteración consta de un incremento en el que se han agrupado las funcionalidades a implementar.

Por lo tanto cada tarea se corresponde al desarrollo de una funcionalidad, una funcionalidad por rama que se integra en la rama de la iteración que da paso al siguiente proceso...

Continuous Integration

Proceso de Integración Continua.

... el Proceso de Integración Continua es el encargado de automatizar la integración de los desarrollos por ramas lo más a menudo posible, para así poder detectar fallos en la integración.

Continuous Delivery & Deployment

Proceso de Entrega y Despliegue Continuo

El siguiente paso en el desarrollo Iterativo e Incremental es la entrega, donde entra el Proceso de Entrega y Despliegue Continuo.

Encargado de utomatizar y mejorar el proceso de entregas y despliegues. Prepara la entrega de la solución para diferentes entornos, agilizando y mejorando en cada iteración completada dentro del desarrollo del proyecto.

Resultados

El análisis generó la necesidad de crear un sistema modular que permita integrar estas herramientas del ciclo de vida de un proyecto como vehículo en la forja de desarrollo.

Después de haber analizado las soluciones existentes, se encontraron soluciones privadas o con pocos recursos, por lo que se reafirma la opción construir una forja de desarrollo FLOSS que integre el proceso Iterativo e Incremental, desde la recogida de los requerimientos hasta la construcción de la solución (sistema de construcción integrado).

SCStack

Nueva rueda FLOSS

El conjunto de herramientas apiladas y conectadas que dan forma a la Forja para controlar el ciclo de vida el software.

SCStack aglutina la gestión de las tareas, el código fuente, la orientación del desarrollo a los tests la integración, las entregas y despliegues continuos.

Aprovisionamiento

“Accción o efecto de aprovisionar, aprovisionar : abastecer.“

Aplicado a la Ingeniería del Software el Aprovisionamiento nos provee de los componentes necesarias para construir una solución.

El aprovisionamiento trata de la automatización de tareas para construir el entorno deseado, en este caso la instalación de SidelabCode Stack. La evolución en el modelo de instalación para facilitar la ejecución de módulos, configuración y comunicación entre los distintos componentes.

Puppet


exec { "apt-update":
    command => "/usr/bin/apt-get update",
}
class { "scstack":
    # Superadmin password. Will be used to access the SidelabCode Stack Console *******
    sadminpass => "*******",
    # Or whatever IP specified in Vagrantfile
    ip => "192.168.33.10", 
    domain => "sidelabcode.scstack.org",
    baseDN => "dc=sidelabcode,dc=scstack,dc=org",
    # Your company/organization name
    compname => "SidelabCode Stack version 0.4",
    # A name to be displayed within Redmine
    codename => "SCStack ALM Tools",
}

Puppet: es una herramienta Software Libre de aprovisionamiento desarrollada en Ruby.

Gestiona la infraestructura a través de su ciclo de vida, desde el aprovisionamiento y la configuración de parches automatizando la ejecución de órdenes para la instalación y configuración del entorno.

  • Automatizar tareas repetitivas.
  • Desplegar rápidamente aplicaciones críticas.
  • Gestionar proactivamente el cambio.
  • Escalar servidores.
  • Instalación local o remota.

Esta es la configuración de Puppet para la instalación de la forja SCStack en un servidor, simple y sencilla: un usuario, una ip, un dominio y un nombre. Barreras 0.

¿ Hemos llegado ya ?

El proyecto gira en torno a un objetivo conciso:

“Crear un marco de trabajo para Integrar el uso de la metodología Iterativa e Incremental a través de las herramientas necesarias“

A partir de la elección de la metodología Iterativa e Incremental se trasladan los requerimientos como parte inicial del proyecto para dibujar el camino deseado a seguir.

Se han incluido las herramientas necesarias dentro del proyecto SCStack. Además su integración dentro del flujo de trabajo y la interoperabilidad entre las mismas dentro del entorno SCStack. Se ha unificando el proceso de desarrollo que aglutina a todos los componentes involucrados mediante la configuración y la accesibilidad de las herramientas dentro de la forja creando un marco de trabajo.

Se traza el camino a seguir para introducir la metodología en el mismo proceso de desarrollo aprovechando las cualidades de cada una de las herramientas. Se localizan y gestionan los recursos necesarios para crear el marco de trabajo Iterativo e Incremental.

Documentando el proceso de desarrollo para que a través del marco de trabajo creado se facilita su integración y uso por parte de los actores implicados.

Se ha comprobado a través de casos reales el correcto funcionamiento del proceso de desarrollo a través de SCStack. Con es el ejemplo de la instalación en la empresa TSCompany.

Por último, se ha dotado a la forja de un potente sistema de inicialización y configuración a través de Puppet, permitiendo la incorporación de futuras herramientas, actualizaciones de versiones o sustituciones de módulos que pueden aportar mejoras.

Se ha conseguido integrar de forma satisfactoria el proceso de desarrollo Iterativo e Incremental en el marco de trabajo de SCStack

Siguientes Iteraciones

Como parte del proceso iterativo e incremental, las siguientes iteraciones han de aportar valor:

  • Refactorización del código
  • Publicitar el uso de la herramienta
  • Modularizar la instalación

Refactorización del código mejorando el rendimiento, la eficacia y la facilidad de evolución.

Mover a la comunidad facilitando el acceso a la herramienta, la difusión y el uso. La barrera de entrada es baja debido a la facilidad para la puesta en marcha, el uso de herramientas conocidas FLOSS y la ayuda de la documentación.

Con respecto a las funcionalidades y herramientas de SCStack, se ha de centralizar la instalación de los módulos, es decir, dotar al usuario de una herramienta para que él mismo elija los componentes de SCStack que quiere instalar en su servidor.
