Que, porqué y para qué

El desarrollo de este proyecto se basa en el diseño e implementación del proceso de desarrollo acorde con las metodologías ágiles a través de la forja SidelabCode Stack. Unificando el comportamiento de las herramientas a través de las distintas APIs, facilitando la instalación, la replicación y la recuperación de los datos mediante el uso de Software Libre para construir Software de calidad.

¿ Que es sideladCodeStack ?

¿ Que es sideladCodeStack ? SCStack una forja de Desarrollo, una herramienta ALM para controlar el ciclo de vida de una aplicación.

¿ Por qué aparece ?

¿ Por qué aparece ? Por la necesidad de centrar la atención en el desarrollo facilitando la composición del entorno estableciendo unas herramientas para guiar el proceso con buenas prácticas.

¿ Para que sirve ?

¿ Para que sirve ? Para ayudar a construir software dentro de un entorno controlado que facilita la evolución del mismo siguiendo unas pautas.

La necesidad de tener un esqueleto para el desarrollo de un proyecto a través del uso de herramientas de Software Libre.

Nube de forjas

Nube de forjas, lista, etc, diapositivas verticales... Partiendo del principio DRY (Don’t Repeat Yourself) y de la mano de No reinventar la rueda se analizaron las forjas ALM existentes.

Github, Bitbucket, Collab.net, Cloudbees, Plan.io, ClinkerHQ, Google Code, Fedora Hosted, BerliOS, Gitorius, GNU Savannah, Gna!, CodePlex, CodeHaus, CloudForge, caen eliminadas para una posible solución.

Análsis 2a iteración

Requisitos para adapatar el proceso Iterativo e Incremental:

  • Wiki
  • FTP
  • Code-review
Alioth y GitlabHQ se descartan. En el último escalafón tenemos a GForge que en 2009 pasó a ser FusiónForge. FusionForge no soportaba repositorios distribuidos como es el caso de Git en sus primeras versiones.

Resultados

El análisis generó la necesidad de crear una forja de desarrollo Libre y usable en distintos entornos.

Después de haber analizado las soluciones existentes, se encontraron soluciones privadas o con pocos recursos, por lo que se reafirma la opción construir una forja de desarrollo FLOSS que integre el proceso Iterativo e Incremental, desde la recogida de los requerimientos (ITS) hasta la construcción de la solución (sistema de construcción integrado).

SCStack

Nueva rueda FLOSS

El conjunto de herramientas apiladas y conectadas que dan forma a la Forja.

Tras analizar las soluciones ALM existentes se optó por crear una nueva Forja a partir de las herramientas necesarias para el proceso de desarrollo de Software de Calidad para adaptar la metodología ágil iterativa e incremental. Para facilitar y unificar el proceso de desarrollo se optó por crear una herramienta que aglutinase la gestión de las tareas, el código fuente y la orientación del desarrollo a los tests.
